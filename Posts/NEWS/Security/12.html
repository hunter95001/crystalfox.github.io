AI를 활용한 악성코드 탐지 - 고려대 김휘강 교수 [REAL2019 - 클라우드&보안]

Security

작성일 : 2020-03-30

본 영상은 삼성 SDS에 특강내용인 AI를 활용한 악성코드 탐지에 관해서 고려대 김휘강 교수님이 발표하신 자료입니다.

악성코드를 탐지하는 시스템을 만들때 고려하는 사항
A.I를 기업에서 사용하기 위해서는 정보손실이 없도록 원시로그를 사용해줘야함
웹로그, 리눅스 syslog, 윈도우 이벤트 로그나 CPU 메모리, 네트워크 플로우데이터를 직접 핸들링 해야함

aps에서 필터링된 로그를 가지고 더 무언가를 해보려 하기 때문에 사람의 Roll Base를 가지고 필터링된 데이터를 가지고 Detection을 하려고 하기 때문에 유추가 제대로 되지 않음
어떤 데이터를 더 가공하고 로그 디자인을 공격적으로 해야만 탐지율을 높일것인가

suspicion한 이벤트가 발생할경우 네트워크 관리자나 서버관리자가 IPsec을 걸거나 네트워크장비에서 ACL을 걸어서 서버를 Blocking해야만 했지만 Firewall이 이를대신하고있다

대채불가능 (전문가들의탐지룰 <-AI튜닝)

Classical A.I -> 랜덤포레스트, 디시전트리, 역전파 기존의 보안솔루션에 사용중임, 한계가 있다.
커다란 데이터를 [String Line으로 계속 들어오는 데이터] Deep Convolution한 thinking을 할 수 있는 객채를 만들어야함

GAN은 노말한 데이터를 이용해서 공격데이터를 만들 수 있다.
1. 자율주행 자동차 보안
- 일반적으로 99% 노말함 1%가 공격이 들어올까말까함 [이때 적용하기 좋은게 GAN]
2. 공장같은 SCADA 시스템 제어시스템 네트워크 보안

각 도메인에 맞는 알고리즘을 찾아서 적용해야함 무조건 CNN이나 DNN같은 딥러닝 알고리즘이 좋은것이 아님
악성코드의 경우 CNN이나 DNN을 사용할경우 신종공격을 만들어서 공격할경우 정확도가 높지않은 모습을 보여줌.

1.사람 - 머신러닝은 블랙박스검사 여서 탐지는 잘 되지만 어떻게 탐지됬는지 모른다
2.양질의 데이터가 부족.
3.머신러닝으로 만든 디텍션 알고리즘을 공격을 하는 adversarial machine Learning attack이 화두다
[사람이 볼땐 별차이가 없는데 기계가볼땐 큰 차이가 나는 노이즈를 강제로 주입함]
4.Scalability함 -> 데이터가 100, 1000, 10^n개 있을때도 scalability한 응답을 얻을 수 있는지
5.privacy
