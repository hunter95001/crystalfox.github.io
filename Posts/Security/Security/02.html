사이버 보안 빅데이터와 머신러닝

1. 빅데이터 개요
1.1 빅데이터란
- 빅데이터는 큰 용량, 바른속도, 다양성이 높은 정보 자산이다. 이것으로 의사결정 및 통찰 발견, 프로세스 최적화를 향상시키려면 새로운 형태의 처리 방식이 필요하다.
- 3V : 규모(volume), 빠른 유입 속도 (velocity), 다양성(variety)
- 4V : + 정확성 (Veracity) <- 높은 데이터 품질 확보가 필요
- 5V : + 가치 (Value)

1.2 데이터 분석 프로세스
- 목표 수립
- 데이터 수집
- 데이터 저장
- 데이터 처리
- 데이터 모델링 (수동[통계] / 자동 [머신러닝])

1.3 빅데이터와 분석 = 플랫폼 구축 활용 + 데이터 분석 기술
플랫품 구축 활용 : 분석 목적/환경에 맞는 플랫폼을 구축하고 이를 활용하는 능력
데이터 분석 기술 : 데이터를 이해하고 원하는 정보를 뽑아내는 데이터 분석 능력

2. 사이버 보안 빅데이터
2.1 사이버보안 빅데이터란?
- 사이버보안 분야에서 생성되고 활용되는 빅데이터로, 악성코드, 로그, SNS 등 다양한 유형의 데이터를 포함
- 일반적인 빅데이터와 데이터 속성 자체는 크게 다르지 않음
- 오래 전부터 연구 대상이었으나, 최근에 와서야 실질적인 활용이 되고 있음
- 분야의 특성상 개인이 데이터를 수집, 확보하는 것이 쉽지 않음
* 양 질의 데이터를 구하기가 힘들다.

2.2 사이버보안 빅데이터 활용 사례
2.2.1 악성코드 탐지
- 아무리 복잡한 난독화 알고리즘 구조를 가진 악성코드도 결국에는 목적 코드를 실행
- 샌드 박스 기반 행위 분석 후 수집한 정보로 악성코드를 분류하거나 특성 파악 가능
- 최근에는 딥러닝 기반 악성코드 탐지 연구가 활발히 진행되고 있음

2.2.2 네트워크 (호스트) 보안 위혐(이상) 탐지
- 초기에는 네트워크 분야에서만 적용되었으나 최근에는 호스트 기반 정보도 함께 적용
- 대부분의 경우 네트워크 트래픽 모델을 기반으로 하는 네트워크 비정상 행위 탐지 방식(anomaly detection) 을 사용
- '정상'으로 간주되는 트래픽의 분포를 분석해 새로운 데이터가 정상 데이터에서 얼마나 벗어나 있는지(outlier) 탐지
- 충분한 양의 데이터 + 기존 시그니처 방식의 탐지 기법과 함께 사용해야 함

2.2.3 침해사고 분석 [증거나 데이터를 빅데이터 분석 기법을 토대로 효율적이고 체계적으로 분석을 하고 시간을 단축]
- 파일 카빙 과정에서 대상 파일의 그니처가 조각나 있는 경우 (모델링을 통해 파일 확장자를 찾아낼 수 있음)
- 방대한 양의 데이터에서 디지털 증거를 수집해야 하는 경우 (데이터 정제로 부하를 줄인 후 빠르고 정확하게 판단)
- 이벤트 로그 분석으로 침해사고 여부를 찾아내 하는 경우 (일정한 패턴을 가진 로그로 잠재적 위협 판단 가능)

2.2.4 정보보호정책
- 대부분 업무가 복잡한 계산보다는 경험과 사람의 판단이 중요시 되는 정책 분야
- 사례 : 웹 사이트에서 제시하는 개인 정보 사용 동의서와 관련한 연구
= 일반적으로, 회원 가입 또는 개인 정보 제공이 필요한 서비스 이용 시 개인정보 활용 범위와 법적 근거에 대한 장문의 안내서를 제대로 읽지 않음
= 설령 꼼꼼히 읽는다하여도 의미를 이해하기는 어려움
= 프라이버시 정책 문서를 자동으로 분석해 사용자에게 시각적으로 보여줄 수 있다면?

2.3 사이버보안 빅데이터센터 소개
2.3.1 빅데이터센터 개요
- (KISA) 빅데이터 분석을 통해 침해사고 대응역량 강화
- (산학연) 위협 빅데이터와 분석 플랫폼을 제공하여 제품개발 및 연구 지원
- 센터에 방문해서 데이터 신청을 하면 데이터를 허가해줌
- 분석용 가상머신을 제공 [가상 머신 안에 데이터가 들어가 있음]
