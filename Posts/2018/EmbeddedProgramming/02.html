IoT 시스템 아키텍처 구성

데이터 생성/소비. D(Device)
연결 Connection 6(세션),7(애플리케이션) layer에서 통신. N(Network)
데이터 처리, 빅데이터 처리. P(Processing)
서비스 제공. C(S)

IoT 장치 예

온습도 측정 장치
미세먼지 측정 장치
움직임 측정 장치
영상 입력 장치 -운전자 졸음 시 알람
IoT 콘센트 - 소모되는 전력량과 원격으로 on, off를 제어 가능합니다.
커튼 제어 센서
가전 장비 -세탁기 등.

IoT 장치의 주요 기능

데이터 수집 - 환경요인 센싱 및 데이터 생성
데이터 소비 - 환경요인에 대한 액추에이팅(제어)

데이터 통신 - 원격 호스트로 데이터 전송 또는 제어 명령 수신

IoT장치 -기본 구조

Controller
-Sensors (수집)
-Actuators (제어)
-comm (통신)

프로그래밍 -> 데이터 처리
임베디드 프로그래밍 -> 데이터처리 & 신호처리 [Sensors <=> Controller (2진수가아닌 전기 신호) ]

전기 신호

1. 디지털 신호 -> 두가지 상태값을 가지는 신호 [LOW ,HIGH]
2. 아날로그 신호 -> 신호값이 다양 함 ADC/DAC (아날로그 <=> 디지털 변환기.)

Controller

Controller장치는 주로 MCU기반으로 구현
Controller장치는 시리얼 통신방식으로 데이터 통신
Controller장치는 크기가 작아야한다.

IoT 프로그래밍

간단한 알고리즘을 기반으로 프로그래밍하여야 한다.
주로 외부 장치와의 인터페이스 기능을 구현한다.
필요에 따라 인터럽트 기반의 비동기 동작을 구현한다.
-디지털 신호 입출력
-아날로그 신호 입출력
-시리얼 데이터 통신
