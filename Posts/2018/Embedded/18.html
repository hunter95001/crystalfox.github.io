 시리얼 통신 프로토콜
 I2C(Inter-Integrated Circuit) -동기식
 SPI(Serial Peripheral Interface) -동기식
 UART(Universal Asynchronous Receive-Transmit) -비동기식


비동기식 동기식
거리 1M~4M 1M이하 (노이즈가 많이 걸림.)
전송속도 느림 빠름

UART


대표적인 비동기 시리얼 통신
통신속도만 약속으로 맞춰주고 나머지는 디폴트값 =빨간색

RS-232식


20M이상 되는 거리를 연결 하기 위해 신호를 증폭함 (UART는 안됨.)
컴퓨터에서는 UART를 이용하지만 단말기는 RS-232식 포트가 나와있어서 그걸 사용했다.

I2C(II2)


동기식 (Master [Clock을 생성함] -Slave)
칩과 칩사이의 통신을 이야기함 (IC <-> IC)
모든 데이터 전송을 유발하는 동작을 Master가 함 -모든 동작은 MASTER
하나의 장치에 127개까지 연결함.
[I2C장치들은 각각의 주소를 가지고 있다. -> 제조사가 주소를 할당함(제조사마다 주소가 다름 ) =장치가 충돌 날 수가 있다.]
RX-TX 가 없고 하나로 만들어짐.
pull duplex - 송신 수신이 같이 일어남.
half duplex - 송신하거나 수신 하나만 일어남.
간단한 통신 프로토콜.
장거리 통신 불가능 (2~30cm)


I2C장치에 풀옵 저항을 달아야함.

// I2C Scanner
// Written by Nick Gammon
// Date: 20th April 2011
 
#include <Wire.h> //기본적으로 제공해주는 라이브러리 
 
void setup() {
  Serial.begin (9600);
  Serial.println ("I2C scanner. Scanning ...");
  byte count = 0;
  
  Wire.begin();
  for (byte i = 8; i < 120; i++) //0~7번 121~127번은 잘 안 쓴다. 호스트.
  {
    Wire.beginTransmission (i); //주소를 받는다
    if (Wire.endTransmission () == 0) //endTransmission을 종료
      {
      Serial.print ("Found address: ");
      Serial.print (i, DEC);
      Serial.print (" (0x");
      Serial.print (i, HEX);
      Serial.println (")");
      count++;
      delay (1);  // maybe unneeded?
      } // end of good response
  } // end of for loop
  Serial.println ("Done.");
  Serial.print ("Found ");
  Serial.print (count, DEC);
  Serial.println (" device(s).");
}  // end of setup
 
void loop() {}// I2C Scanner

#include <Wire.h> 
#include <LiquidCrystal_I2C.h>
 
// Set the LCD address to 0x27 for a 16 chars and 2 line display
LiquidCrystal_I2C lcd(0x3F, 16, 2);
 
void setup()
{
    // initialize the LCD
    lcd.begin();
 
    // Turn on the blacklight and print a message.
    lcd.backlight();
    lcd.print("Hello, world!");
}
 
void loop()
{
    lcd.setCursor(0,1);
  lcd.print(millis()/1000);
  delay(1000);
}
