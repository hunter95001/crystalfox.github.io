2018-10-08 클래스다이어그램


클래스 다이어그램은 문제나 해결책의 정적인 구조를 보여주는 매우 중요한 다이어그램이다.
클래스 다이어그램은 시스템을 구성하는 클래스들과 그들간의 관계를 보여준다.

클래스 이름
속성
오퍼레이션으로 구분

속성이름 : 속성타입
접근 제어자 표시 설명
Public + 모든 클래스 적용가능
Private - 클래스 안에서만 가능
Protected # 클래스 안에서만 가능

- id :String
- 주소 :String
+ 연락처 :String

제약조건
{}나 노드심볼 을 이용하여 기술
salay : int{salary >0}
관계 설명
연관 관계(160Page) 선으로 그은것. (역할, 연관관계이름,다중성)
일반화 관계 상속을 나타냄
의존 관계 잠시 객체가 생겼다 사라질 경우
집합 관계 (160Page) 고급 클래스를 만들때, 집약, 합성
실체화 관계 (160Page) 인터페이스를 표현할 때 -공통속성이 있을때.

다중성표기 의미
1 1
* =0..* 0또는 그이상
1..* 1이상.

양방향 화살표로 표시 없음
단방향 화살표로 표시

일반화관계 -상속
집약 =빈마름모
-전체와 객체의 부분이 종속적일떄
합성 =채워진 마름모
-전체와 객체의 부분이 독립적일떄

다중상속 -> 점선과 세모.

Class와 Interface를 이용해서 그리면 됨.

Toolbox를 이용하면 노트를 이용할수 있음

관계 설명할려면 연관선을 클릭하면 됨.
Attributes 숨기기 Format ->Suppress Attributes
Operations 숨기기 Format ->Suppress Operations
모양 바꾸기 Format ->Stereotype Display-> textual[사각형] Iconic [원]
