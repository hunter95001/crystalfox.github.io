테스트는 시스템을 개발할 때 요구되는 동작과 실제 개발한 후의 동작이 다르지는 않은지 분석하고 확인하는 작업이다.

100% 테스트는 불가능
- 판단 불가능
- 시간 자원 제약
중요성
- 우주 탐사선 컬럼비아 호
- 많은 경험이 필요

테스트는
-오류가 있음을 보여주려는 작업
-오류가 없음을 보여주는 작업은 아님
테스트 단계
테스트 명 설명
단위 테스트 컴퍼넌트[구성요소, 모듈]를 테스틑 한다
블랙박스 테스트, 화이트박스 테스트, 객체지향 테스트 가 있음.
통합 테스트 컴퍼넌트를 독립적으로 테스트한 후 [단위테스트가 끝난 후] 인터페이스를 테스트하는 것이다.
모듈에서 다른 모듈로 넘어가는 것을 보여줌.
시스템 테스트 기능 테스트 : 기술된 기능이 잘 수행되는지 테스트 하는 것.
성능 테스트
Stress test : 수강 신청에서 많은 트렌젝션을 받아서 수행하거나 게임 서버의 수용인원 등을 테스트함
Volume test : 데이터를 계속 보내서 얼마나 견딜 수 있는지 확인
Security test :보안과 관련된 테스트
인수 테스트 고객에게 프로그램을 넘겨주고 그 프로그램이 재대로 동작하는 확인하는 테스트.
설치테스트 사용자에게 사용할 환경에 시스템을 설치하고 테스트를 정검합니다.
