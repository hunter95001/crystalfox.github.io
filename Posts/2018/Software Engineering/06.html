용자관점을 이해해야 유용한 시스템을 만들 수 있다. (What)
개발자 관점 (How)

유스케이스의 역할 : 사용자의 관점에서 시스템을 모델링

액터: 이벤트 흐름을 시작하는 객체
포함[include] 관계 : 유스케이스를 수행할 때 다른 유스케이스가 드시 수행되는 것 [필수, 공통]
확장[extend] 관계 : 상위 유스케이스로부터 어떠한 특정 조건에 의해 수행됨. [선택]
일반화 관계

모델링 절차

1. 시스템 상황을 확인
2. 액터 식별
3. 유스케이스 식별
4. 유스케이스 다이어그램 작성
5. 유스케이스 명세서 작성
6. 유스케이스 실체화.

관찰

사용자의 업무를 관찰하는 것이야말로 대화에서 놓치기 쉬운 자세한 사항을 파악할 수 있는 방법이다.
관찰하고 결과 정보를 분석하는데에는 시간이 많이 소요된다.

인터뷰시 유의 사항

인터뷰하는 사람은 무엇보다도 고객과 사용자에게 공감하고 잘 듣는 훈련이 되어 있어야 한다.
